# astrbot_plugin_dfhelper

三角洲行动助手 AstrBot 插件。支持实时市场卡战备查询、自定义凑战备查询、今日密码查询和特勤处制造利润查询。

## 安装与配置

1. 将本目录放入 AstrBot 的 `data/plugins/astrbot_plugin_dfhelper`。
2. 在 AstrBot WebUI 插件管理中启用插件。
3. 前往 [API 开通页面](https://orzice.com/work/v/home) 开通 API，并获取 `token`。
4. 在插件配置中填写：
   - `api_token`：在 API 开通页面获取的 `token` 密钥。
   - `timeout_seconds`：接口超时时间，默认 `8` 秒。
   - `result_scheme_limit`：实时卡战备最多展示方案数量，默认 `5` 个。
   - `result_item_limit`：每个方案最多展示明细数量，默认 `12` 项。
   - `trade_match_limit`：交易行物品查询最多展示匹配数量，默认 `8` 项。
   - `item_price_cache_seconds`：交易行价格缓存时间，默认 `45` 秒。
   - `user_whitelist` / `user_blacklist`：用户白名单和黑名单，填写用户 ID。
   - `group_whitelist` / `group_blacklist`：群白名单和黑名单，填写群 ID。
   - `admin_user_ids`：管理员用户 ID，可为用户添加和查询功能使用次数。

资格规则：黑名单优先拒绝；管理员、用户白名单、群白名单免次数；其他用户需要对应功能剩余次数，接口成功返回后扣减 1 次。


## 指令

### 帮助

发送：

```text
/三角洲帮助
```

别名：

```text
/df_help
/df_bz
```

插件会列出所有可用命令、别名和常用示例。

### 次数管理

管理员添加次数：

```text
/添加次数 <用户ID> <功能> <次数>
```

别名：`/tjcs`。

查询次数：

```text
/查询次数 [用户ID]
```

别名：`/cxcs`。

普通用户只能查询自己；管理员可查询任意用户。次数可以为负数，用于扣减用户剩余次数，但不会低于 `0`。

功能参数：

| 功能 | 参数 |
| --- | --- |
| 卡战备 | `kzb`、`卡战备` |
| 自定义卡战备 | `zdykzb`、`自定义卡战备` |
| 今日密码 | `jrmm`、`今日密码` |
| 制造利润 | `zzlr`、`制造利润` |
| 交易行查询 | `jyh`、`交易行` |

示例：

```text
/tjcs 123456 kzb 10
/tjcs 123456 zzlr -2
/tjcs 123456 jyh 10
/cxcs 123456
```

### 实时卡战备

发送：

```text
/卡战备
```

插件会返回数字参数并等待用户回复：

```text
0：11W 机密配置
1：18W 机密配置
2：55W 绝密巴克什
3：60W 绝密航天
4：24W 适应监狱（每周五12:00 ~ 周一00:00 开启才计算）
5：78W 绝密监狱
```

用户在 60 秒内回复 `0` 到 `5` 后，插件调用 `/v1/sjz_api/jzv3_zb` 查询并返回结果。回复 `取消` 可退出。

别名：`/kzb`。

### 今日密码

发送：

```text
/今日密码
```

别名：

```text
/密码
/地图密码
/保险密码
/jrmm
```

插件会调用 `/v1/sjz_api/map_pwd` 查询今日地图密码。地图对应关系：

| 字段 | 地图 |
| --- | --- |
| `a` | 零号大坝 |
| `b` | 长弓溪谷 |
| `c` | 巴克什 |
| `d` | 航天基地 |
| `e` | 潮汐监狱 |

值为 `-` 表示暂未更新，接口每天凌晨 `0:05` 更新。

### 制造利润

发送：

```text
/制造利润 <部门> <部门等级>
```

别名：`/zzlr`。

插件会调用 `/v1/sjz_api/manufacturePro` 查询特勤处制造利润，并按每小时收益从高到低展示。

部门参数支持中文名或拼音首字母缩写：

| 部门 | 缩写 |
| --- | --- |
| 技术中心 | `jszx` |
| 工作台 | `gzt` |
| 制药台 | `zyt` |
| 防具台 | `fjt` |

部门等级：`1`、`2`、`3`。

示例：

```text
/制造利润 技术中心 3
/zzlr jszx 3
```

### 交易行物品查询

发送：

```text
/交易行 <物品名称关键词>
```

别名：

```text
/查物价
/物价
/jyh
```

插件首次加载后会请求 `/v1/sjz_api/item_info_all` 获取基础信息库，并保存到插件目录的 `item_info_cache.json`。后续启动如果本地缓存有效，会直接使用本地缓存，不再重复请求基础信息库。

查询时插件会按名称关键词匹配本地基础信息库，再调用 `/v1/sjz_api/item_price_all` 获取交易行价格数据，并通过基础信息库的 `oid` 与价格库的 `id` 关联。价格库会按 `item_price_cache_seconds` 短时间缓存，避免连续查询重复请求。

示例：

```text
/交易行 Vector
/jyh DAS
```

返回内容包括物品名称、分类、品质、价格、更新时间和交易行 ID。

### 自定义卡战备

发送：

```text
/自定义卡战备 <战备值> <需求/已有> <装备类型...>
```

别名：

```text
/自定义战备
/diy卡战备
/zdykzb
```

战备值范围：`1500` 到 `780000`。

模式说明：

- `需求`：请求时只将列出的装备类型设为需要，其余装备类型关闭。
- `已有`：请求时将列出的装备类型视为已有并关闭，其余装备类型设为需要。
- `兑换` 和 `双枪` 不参与选择，所有模式下都固定关闭。

装备类型：

| 装备 | API 参数 | 说明 |
| --- | --- | --- |
| 背包、bb | `is_bb` | 是否启用背包 |
| 枪、枪械、qx、gun | `is_gun` | 是否启用枪械和配件 |
| 护甲、甲、hj | `is_hj` | 是否启用护甲 |
| 手枪、sq | `is_sq` | 是否启用手枪 |
| 头盔、盔、tk | `is_tk` | 是否启用头盔 |
| 胸挂、xg | `is_xg` | 是否启用胸挂 |

模式别名：`xq` 表示 `需求`，`yy` 表示 `已有`。

## 使用示例

```text
/自定义卡战备 78000 需求 胸挂 背包
/自定义卡战备 78000 已有 胸挂 背包
/zdykzb 16000 xq tk xg
```

返回示例：

```text
卡战备查询结果
目标战备：500,000
方案名称：自定义
总战备：501,360
总价格：451,070
差值：-48,930
配装明细：
- 枪械1：Vector冲锋枪 | 价格 140,795 | 战备 11,158 | 等级 1
- 头盔：DAS 防弹头盔 (几乎全新) | 价格 25,188 | 战备 -2,974 | 等级 3
```

## 错误处理

插件会处理以下错误并向用户返回明确提示：

- 未配置 `api_token`。
- 战备值不是整数或超出 `1500` 到 `780000`。
- 实时卡战备数字参数不是 `0` 到 `5`。
- 自定义卡战备模式不是 `需求` 或 `已有`，或装备类型不存在。
- 今日密码返回字段缺失或结构异常。
- 制造利润部门不是支持的中文名或拼音首字母缩写，或部门等级不是 `1` 到 `3`。
- 交易行基础信息库未加载、物品关键词过短或没有匹配结果。
- 网络异常、接口超时、HTTP 非 200、接口返回错误码、返回 JSON 结构异常。
